## Proposed CLI surface (verbatim â€” subcommands + key flags)

```text
cyclerfinder [--version] [-v|--verbose] <subcommand> ...

# 1. enumerate â€” list cells + Tisserand feasibility for a body set
cyclerfinder enumerate
    --bodies V,E,M              # comma-separated body codes (default: V,E,M)
    --l-max 6                   # max encounters in the sequence (default: 6)
    --k-max 3                   # max period in synodic/beat multiples (default: 3)
    --n-max 0                   # max revolutions per leg (default: 0)
    --branch single             # comma-list: single|low|high (default: single)
    --vinf-cap 7.0              # km/s feasibility cap (default: 7.0)
    --feasible-only             # only emit cells passing tisserand_feasible
    --period beat               # 'beat' (natural beat, period_basis=None) |
                                #   anchor pair 'E-M' (sets period_basis)
    --format table|json|csv     # output format (default: table)
    --limit N                   # cap rows emitted (default: no cap)

# 2. solve â€” optimise a single cell at a fidelity/mode
cyclerfinder solve
    --cell-id 'VEM|E-M-E-E-V-E|k3|r00000|bsssss|pEM'   # OR build via flags:
    --bodies V,E,M --sequence E-M-E-E-V-E --k 3 --period-basis E-M
    --revs 0,0,0,0,0 --branch single,single,single,single,single
    --fidelity idealized|ephemeris        # (default: idealized)
    --mode maintenance|ballistic          # ephemeris only (default: maintenance)
    --vinf-cap 7.0
    --n-starts 5 --seed 0 --no-de
    --priority-date 2032-01-01            # ephemeris epoch resolution
    --vinf-targets E=5.65,M=3.05          # ephemeris phase-match targets
    --format table|json

# 3. discover â€” run the ledger-backed discover() loop with V-level gates
cyclerfinder discover
    --bodies V,E,M --k 3 --vinf-cap 7.0
    --ledger out/vem.jsonl
    --l-max 4 --n-max 0 --branch single
    --max-cells N
    --fidelity idealized|ephemeris        # -> discover(optimiser=...)
    --enable-v3                           # ballistic-closure V3 gate (M-ED)
    --priority-date ... --vinf-targets ...
    --n-starts 5 --seed 0 --no-de
    --format table|json

# 4. report â€” campaign summary from the ledger (+ gauntlet verdicts)
cyclerfinder report
    --ledger out/vem.jsonl
    --out out/vem-report                  # writes .md and .json
    --format md|json|both                 # (default: both)
    --with-verdicts                       # attach run_gauntlet() tiers

# 5. viz â€” visualisations for a VEM cell (requires the [viz] extra)
cyclerfinder viz porkchop  --bodies E,M --tof-min 100 --tof-max 400 \
    --epoch-range 2032-01-01:2039-01-01 --out out/porkchop.png
cyclerfinder viz trajectory --cell-id '...' --fidelity idealized \
    --out out/traj.png
cyclerfinder viz beat --bodies V,E,M --out out/beat.png
```

VEM defaults (`--bodies V,E,M`, `--period beat`, `--vinf-cap 7.0`) realise the
M8 anchor with zero required flags: `cyclerfinder enumerate --feasible-only`
and `cyclerfinder discover --ledger out/vem.jsonl` both run the VEM campaign
out of the box.

## Resolved design decisions

1. **argparse, not click/typer.** Stdlib, zero new deps, and the repo already
   uses argparse (`data/catalog.py:1021`). Typed handlers + per-subcommand arg
   builders keep it readable.
2. **matplotlib stays the optional `[viz]` extra.** Already declared; lazy-
   imported; Agg + file-output in tests; base CLI never imports it; missing
   extra â†’ exit 3 with install guidance.
3. **`report` enforces sourced âŠ¥ computed at the data-model level** (disjoint
   key sets), not just by convention â€” so the golden discipline is structural,
   not a reviewer's vigilance.
4. **Ballistic / V3 exposed-but-inert.** The CLI surface promises the M8
   ballistic capability, but its convergence is M-ED's deliverable; M8-UX ships
   the plumbing + a documented verify-step rather than blocking on or faking
   convergence.
5. **One existing-file edit only** (`pyproject.toml` `[project.scripts]`). All
   other targets are new files, respecting the concurrent-edit fence on
   `src/`, `tests/`, `data/`, and the site.
