## 0. Verdict — USEFUL (high)

**The unique value of this report is Appendix E: ~47 pages of machine-printed test
cases with complete inputs AND outputs, in canonical Earth units (DU/TU, mu=1), for
exactly the routines we implement: Gauss/Lambert (16 cases), universal-variable
Kepler propagation (14 cases), RV<->elements (20+ cases), Gibbs / Herrick-Gibbs,
Stumpff C&S functions, Kepler's-equation iteration, and a Hohmann interplanetary
table.** Many cases additionally cite their origin in Bate/Mueller/White (BMW)
*Fundamentals of Astrodynamics* appendix/problem numbers, giving double published
sourcing. The 4th-edition book does **not** publish per-iteration traces; TR-91-6
prints the full iteration table (z, y, x, t_n per step) for every Gauss case —
uniquely useful for debugging our solver's convergence path, not just its endpoint.

Negative findings (record so we never re-mine):

- **Cyclers / free-return / Earth–Mars round trips: ABSENT.** Closest content is a
  Hohmann Earth-to-planet table (p. E-36) and circular-orbit rendezvous phasing
  (p. E-37). Nothing on resonant returns, flybys, or multi-body trajectories.
- **Multi-rev Lambert: ABSENT — explicitly out of scope.** The GAUSS routine
  narrative (p. 20) states it is "valid for ellipses LESS THAN one revolution,
  parabolas, and Hyperbolas".
- **Differential correction / shooting: ABSENT.** Perturbation section is Cowell +
  RK4 propagation and J2/drag secular rates only (pp. 27–29, E-43 … E-47).
- **One printed-value defect found:** the Mars row of the Interplanetary table
  (p. E-36) is internally inconsistent — see §6. Do not use that row as a golden.

Accuracy caveats stated by the author himself: answers were assembled from a
variety of sources and "should not be considered an absolute guarantee of the
correctness of the algorithm" (Abstract); cross-machine floating-point differences
appear "usually in the 5th or 6th decimal place" (p. iv). So: use these as goldens
with **~1e-5 relative tolerance**, not 1e-12.

## 2. GAUSS (Lambert, universal variables) test cases — Appendix E pp. E-20 … E-27

Algorithm (pp. 20–21): universal-variable Gauss problem solved by **bisection on
z** with initial bounds upper = 4*pi^2, lower = -4*pi, z0 = 0 (C=1/2, S=1/6);
A = dm*sqrt(r1*r2*(1+cos dnu)), dm = +1 short way / -1 long way; y readjustment of
the lower bound when A > 0 and y < 0; convergence |t - t0| < 0.00001 TU
(author notes 0.001 converges in ~10 instead of ~25 iterations but differs in the
3rd–4th decimal; he kept 1e-5); f = 1 - y/r1, g = A*sqrt(y/mu), gdot = 1 - y/r2,
v1 = (r2 - f*r1)/g, v2 = (gdot*r2 - r1)/g. Single-rev only.

All vectors (i, j, k) in DU, velocities DU/TU, times TU; angles deg. "mag" =
printed magnitude. Each case also prints the full per-iteration (z, y, x, t_n)
table — transcribe from the report if needed for trace-level debugging; final z is
given below.

## 3. KEPLER (universal-variable propagation) test cases — pp. E-13 … E-19

Algorithm (pp. 18–19): universal-variable X with Newton iteration on t;
first guesses x0 ≈ sqrt(mu)*(t-t0)*alpha (ellipse; flagged unreliable when
alpha = 1.0 exactly), parabolic guess via cot(2s) = 3*sqrt(mu/p^3)*(t-t0),
hyperbolic log-form guess; **multi-rev guard:** if elliptical (a > 0) and
x_n > 2*pi*sqrt(a), damp the Newton step using 10.0*dt "so the iteration doesn't
converge as quickly"; convergence |t - t_n| < 0.00001. Then f, g, fdot, gdot.

result — a second pair from the J2-perturbed PKEPLER is also printed):

## 6. Interplanetary Hohmann table (p. E-36) — use with care, Mars row is BAD

Setup: 200 km burnout altitude at each planet, mu_Sun = 132715440000.0 km^3/s^2.
Columns: heliocentric distance (km), equatorial radius (km), planet mu
(km^3/s^2), heliocentric v (km/s), vh1 (departure hyperbolic excess at Earth,
km/s), vh2 (arrival excess at planet), vbo (burnout speed at Earth), vret,
transfer days.

Spot-checkable rows (consistent with the classic BMW-era Hohmann numbers):

- Venus: vh1 = 2.5035, vh2 = 2.7160, 145.983 days
- Mercury: vh1 = 7.5343, vh2 = 9.6137, 105.477 days
- Jupiter: mu = 125686537.0, vh1 = 8.7914, vh2 = 5.6431, 996.945 days
- Mars (PRINTED): dist 227 800 000.0 km, r = 3389.9 km, mu = 42828.3,
  vh1 = 4.1745, vh2 = 3.5696, vbo = 11.7735, 311.804 days

**Defect:** the printed Mars outputs are inconsistent with the printed Mars
distance. A Hohmann transfer from 1 AU to 227.8e6 km gives ~2.94 km/s departure
excess and ~259 days, not 4.17 km/s / 311.8 days; the printed outputs are instead
consistent with a Mars distance of ~277.8e6 km — i.e. a digit transposition
(227.8 ↔ 277.8) somewhere in the run that produced the table. (Check used only to
reject the row, per golden discipline.) **Do not use the Mars row as a golden.**
The Venus/Mercury/Jupiter rows agree with their printed inputs and with the
classic textbook values, and remain usable.

## 7. What is unique vs. Vallado's 4th-edition book (why this mining was worth it)

1. **Per-iteration convergence traces** for every Gauss and Kepler case
   (z, y, x, t_n at each bisection/Newton step) — never published in the book.
   Lets us assert not just the answer but the convergence *path* of a
   bisection-based universal-variable Lambert, or diagnose where ours diverges.
2. **Canonical-unit (DU/TU) test set** with one self-consistent constants page
   (p. v) — clean for unit-system-independent testing of `core/lambert.py` /
   `core/kepler.py` with mu = 1.
3. **Deliberate degenerate cases with published expected failures**: collinear
   Lambert (5.11d), near-zero TOF (5.11c), non-coplanar Gibbs, rectilinear
   ellipse, parabolic first-guess — published *rejection* behaviour, which is
   rare and exactly what hardening tests need.
4. The 1991 algorithm differences (bisection-z Gauss vs the book's later
   universal/Battin/Izzo presentations) are documented at implementation grade
   (pp. 18–21), useful context when comparing solver families.
