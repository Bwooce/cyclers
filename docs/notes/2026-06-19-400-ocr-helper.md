# #400 — OCR helper for the Corpus Document Policy

**Date:** 2026-06-19. **Scope:** the *code layer* of the OCR-first rule in
[`corpus-document-policy.md`](corpus-document-policy.md) §1. Makes the OCR
step a single idempotent call any future digest agent reuses, so no corpus
document silently stays a black-box image (the slip that hid Szebehely 1967).

## What was built

- **`src/cyclerfinder/verify/ocr.py`** — `ensure_text_layer(pdf, *, cache_dir,
  min_chars_per_page) -> Path` plus `has_text_layer` and `extract_text`
  conveniences. Probes for an existing text layer (`pdftotext` chars ÷
  `pdfinfo` pages); if image-only, OCRs once with `ocrmypdf --skip-text
  --quiet` into a `cyclerfinder_ocr` astropy-cache subdir (idempotent: a
  cached file newer than the source is reused). Mirrors
  `verify/spice_kernels.py`'s probe → produce-if-missing → cache → return-path
  pattern. The `ocr` extra is optional — nothing from it is imported at module
  load; missing binaries raise a clear ImportError-style message naming the
  extra.
- **`tests/verify/test_ocr.py`** — text-layer-true on a modern arXiv paper,
  image-only-OCRs on a 2-page Szebehely ghostscript sample (`slow`), and a
  missing-PDF guard. All skip cleanly when the OCR toolchain or the private
  corpus is absent.

## Probe threshold — empirical calibration (and a bug it caught)

The initial default `min_chars_per_page = 1.0` was **wrong**. `pdftotext`
emits exactly **one form-feed (`\f`) byte per page** for an image-only PDF, so
such a scan reads 1.000 char/page — landing *on* the `>= 1.0` boundary and
misclassifying as text-bearing. The slow OCR test caught this.

Measured on the corpus (2026-06-19):

| PDF | chars | pages | chars/page | class |
|---|---|---|---|---|
| `szebehely-1967-…-book.pdf` (raw) | 342 | 342 | **1.000** | image-only |
| `braik-ross-2026-…-arxiv-2605.31543.pdf` | 215 409 | 58 | **3713.9** | text layer |

The two populations are ~3 orders of magnitude apart. Default is now
**`10.0`** chars/page — ~10× above the form-feed noise floor, ~370× below real
text, comfortably inside the gap.

**Edge case (honest, not over-engineered):** a scan carrying a *thin
pre-existing* OCR layer (a few words/page) could read just above the floor and
wrongly skip a re-OCR. Rare; for such a document pass a higher
`min_chars_per_page`. We deliberately do not add per-page distribution
analysis — the corpus is cleanly bimodal.

## P400.3 — Szebehely 1967 retrofit (real use-case validation)

Ran `ensure_text_layer` on the one confirmed image-only corpus book.

| | value |
|---|---|
| Source | `cyclers_pdf/papers/szebehely-1967-…-book.pdf` (342 leaves) |
| `has_text_layer` (pre) | **False** |
| chars before OCR | **342** (≈1/page, form-feeds only) |
| OCR runtime | **284 s** (~4.7 min, 342 pages) |
| OCR'd PDF | `~/.astropy/cache/cyclerfinder_ocr/szebehely-1967-…-book.ocr.pdf`, **22.3 MB** |
| chars after OCR | **3 050 571** (~8900/page) |

The OCR'd PDF lives in the astropy cache and is **NOT committed** (large,
binary, regenerable — same rule as `spice_kernels` never committing the BSP).
Re-running `ensure_text_layer` reuses it (idempotent).

`extract_text` now returns readable CR3BP content. Keyword-search samples
(verbatim, Tesseract garble preserved):

- *"the location and properties of the libration points are established"*
- *"…motion in a sy**m**odic coordinate system and the Jacobian integral …
  Equations of motion in dimensionless coordinates"* (synodic → "symodic";
  the equation rows themselves are mangled)
- *"The history of the restricted problem begins with Euler and Lagrange …
  continues with Jacobi (1836) and Hill (1878)"*
- *"…nonlinear stability in the neighborhood of these equilibrium points."*

This confirms the hybrid caveat in practice: prose is fully **navigable**;
equations/subscripts/tables are garbled. Use the OCR text to find the right
chapter/section/page, then Read-tool vision on that page image for exact
sourced values.

## P400.4 — Standard digest-flow call sequence (for future digest agents)

Use the **cheap path by default**; reserve expensive vision for the few pages
where exact values matter:

1. **Ensure a text layer (one call).**
   ```python
   from cyclerfinder.verify.ocr import ensure_text_layer, extract_text
   text_pdf = ensure_text_layer(pdf_path)   # original if it has text;
                                            # OCR'd-and-cached if image-only
   ```
   Modern arXiv/journal PDFs return instantly (already text). Image-only scans
   OCR once (minutes of CPU) then cache forever. No vision-reading of page
   images for bulk text — that is the token trap this closes.

2. **Navigate via `pdftotext` / `extract_text`.**
   ```python
   text = extract_text(pdf_path)            # ensure_text_layer + pdftotext -layout
   ```
   Search this for the TOC, index, and the 3–5 project-relevant
   chapters/sections (books → chapter-summary scope; papers → full read).
   **Never** read a whole textbook page-by-page (the 32 MB tool-result failure
   the policy warns about).

3. **Read-tool vision ONLY on precision pages.** For the 2–3 pages with
   equations / tables / subscripts whose exact sourced values you must quote,
   Read the original page image. Do **not** quote a sourced numeric constant
   from the OCR text (Tesseract garbles math — see the Szebehely samples
   above). This is the policy §1 "Hybrid for precision" rule.

Net: cheap deterministic bulk text everywhere, accurate vision only where it
counts.
